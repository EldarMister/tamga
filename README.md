# Тамга Сервис

Тамга Сервис — CRM-система для полиграфии: заказы, производство, склад, сотрудники, смены, задачи, обучение, зарплата и отчёты.

## Текущий стек

- Backend: FastAPI
- Frontend: React 18 + Vite
- Realtime: Server-Sent Events (`/api/realtime/stream`)
- База данных:
  - SQLite по умолчанию для локального запуска
  - PostgreSQL при запуске через `POLYCONTROL_DATABASE_URL` или `docker-compose`
- Загрузка файлов: локальная папка `uploads/`

Важно:

- основной интерфейс теперь поднимается из `frontend-react/`;
- backend автоматически раздаёт React-сборку из `frontend-react/dist`, если она существует;
- legacy frontend в `frontend/` пока сохранён как fallback на время миграции.

## Что умеет система

- управление заказами и этапами производства;
- расчёт стоимости услуг;
- склад материалов и журнал движений;
- сотрудники и роли: `director`, `manager`, `designer`, `master`, `assistant`;
- кадры: смены, чек-листы, инциденты, штрафы;
- задачи: дневные и недельные;
- уроки: YouTube, фото, отметка просмотра;
- заявки на отпуск и больничный;
- отчёты по заказам, материалам и сотрудникам;
- финансовый дашборд директора и экспорт CSV;
- live-обновления экранов без ручного обновления страницы.

## Структура проекта

- `backend/` — FastAPI, бизнес-логика, роутеры, инициализация БД
- `frontend-react/` — основной React frontend
- `frontend/` — legacy SPA на vanilla JS, используется как fallback при миграции
- `legacy-calculator/` — старый калькулятор
- `uploads/` — загруженные файлы
- `docker-compose.yml` — запуск приложения с PostgreSQL
- `Dockerfile` — сборка backend-контейнера

## Локальный запуск

Рекомендуемый сценарий для полноценного тестирования:

1. Установить Python-зависимости:

```bash
pip install -r backend/requirements.txt
```

2. Установить frontend-зависимости и собрать React:

```bash
cd frontend-react
npm install
npm run build
```

3. Вернуться в корень проекта и запустить backend:

```bash
cd ..
python run.py
```

4. Открыть приложение:

```text
http://localhost:8000
```

Почему так:

- FastAPI раздаёт API и сам frontend;
- если React уже собран, backend будет обслуживать именно его;
- это сейчас основной и самый стабильный путь для тестирования всего приложения целиком.

## Переменные окружения

Поддерживаются параметры:

- `POLYCONTROL_SECRET` — секрет для авторизации
- `POLYCONTROL_DB_PATH` — путь к SQLite базе
- `POLYCONTROL_DATABASE_URL` — строка подключения к PostgreSQL
- `POLYCONTROL_UPLOAD_DIR` — папка для загрузок

Если `POLYCONTROL_DATABASE_URL` не задан, приложение работает на SQLite.

## Docker

Для контейнерного запуска используется PostgreSQL.

Запуск:

```bash
docker compose up --build
```

После этого приложение будет доступно на:

```text
http://localhost:8000
```

## Realtime

В проекте включён realtime через SSE.

Что обновляется в реальном времени:

- заказы;
- задачи;
- объявления;
- склад;
- сотрудники и профиль;
- заявки на отпуск;
- журнал работы;
- кадры;
- обучение.

Канал realtime:

```text
/api/realtime/stream
```

## База данных

Backend поддерживает два режима:

- SQLite для локальной разработки;
- PostgreSQL для продового и docker-запуска.

Инициализация схемы и сидов выполняется при старте приложения.

## Примечание по миграции

Проект уже переведён на React как основной frontend, но часть legacy-кода пока всё ещё сохранена в репозитории для совместимости и fallback-маршрутов. Это нормально для текущего состояния проекта: тестировать нужно именно React-сборку, которую раздаёт FastAPI.
